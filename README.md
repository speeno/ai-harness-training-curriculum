# 기업 AI 교육: 하네스(Harness) · Agent Skills 커리큘럼

## 폴더 구성

| 경로 | 설명 |
| --- | --- |
| `non-developer/` | 기획·운영·영업·행정 등 **코드 작성이 필수가 아닌** 역할용. 스킬을 '업무 매뉴얼', 오케스트레이터를 '워크플로 조립', `_workspace` 등 **폴더 규약**으로 이해하도록 구성합니다. |
| `developer/` | 엔지니어·인프라·데이터 등 **에이전트 정의·스킬 작성·검증**이 필요한 역할용. `SKILL.md`, `AGENTS.md`, Task·서브에이전트, 검증 개념을 포함합니다. |
| `promotion/` | 위 커리큘럼을 외부 안내·제안서·내부 홍보에 활용할 수 있도록 재구성한 **소개 문서와 통합 홍보 자료**입니다. |

각 폴더 안의 `1-day.md`, `3-day.md`, `1-week.md`는 동일한 목차 뼈대(교육 목표, 대상, 표, 상세, 진행 방법, 하네스·스킬 연계, 산출물, 선택 도구 모듈)를 따르되, **기간에 맞게 깊이와 실습 분량**만 조정합니다.

## 비개발자 vs 개발자 한눈에 보기

| 구분 | 비개발자 트랙 | 개발자 트랙 |
| --- | --- | --- |
| **핵심 질문** | "우리 팀 업무를 어떤 순서와 규칙으로 자동화할 것인가?" | "에이전트가 안전하게 도구를 쓰고, 스킬을 재사용·검증하려면 무엇을 정의해야 하는가?" |
| **스킬(Skill)** | 부서 표준 절차·템플릿·금지 사항을 담은 **업무 매뉴얼**에 가깝게 설계 | `SKILL.md` 구조, 트리거 조건, 예시·경계 케이스를 **코드와 함께** 문서화 |
| **오케스트레이터** | 승인 단계·검토 순서·산출물 위치가 보이는 **워크플로** 관점 | 오케스트레이터 전용 스킬, 위임 경계, Task·서브에이전트 호출 규칙 |
| **워크스페이스** | `_workspace` 등 **폴더 명명·배치 규약**으로 산출물 혼선 방지 | 동일 규약 + 리포지토리·브랜치·검증 스크립트와 연계 |
| **검증** | 체크리스트·샘플 산출물 대조 | 스모크 테스트·리그레션·에이전트 실행 로그 검토 등 **검증 개념** |
| **대표 산출물** | 팀 표준 스킬 초안, 폴더 구조 제안서, 1개 업무 시나리오 자동화 설계서 | 스킬 패키지 초안, `AGENTS.md` 포인터 설계, Task 분해 시나리오 |

## 문서별 빠른 선택 가이드

- **반나절~1일 도입 워크숍**: 각 트랙의 `1-day.md`
- **팀 표준을 잡고 파일럿까지**: `3-day.md`
- **사내 거버넌스·템플릿·파일럿 검증까지**: `1-week.md`

회사별로 이미 쓰는 도구(티켓·문서·승인 시스템)가 다르면, 각 문서 하단 **선택 도구 모듈**에 맞춰 채워 넣으면 됩니다.

## 사전 준비와 이 README의 연결

교육 전에 **Cursor(또는 동등 IDE) 설치**, **회사 정책상 허용된 모델·API 범위** 확인, **연습용 비식별 샘플 데이터** 준비만 되어 있으면 각 기간 문서의 실습을 바로 진행할 수 있습니다. 세부 모듈별 준비물은 `non-developer/` · `developer/` 각 파일의 **진행 방법** 절을 따르십시오.
