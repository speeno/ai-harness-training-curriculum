# 개발자 트랙 1주 과정 소개

## 한 줄 소개

사내 하네스 표준을 코드와 문서로 동시에 확립하고, 스킬 재사용·오케스트레이션·검증 파이프라인·운영 런북까지 설계하는 개발 리더 과정입니다.

## 누구에게 맞나요

- 아키텍트, 시니어 엔지니어, 플랫폼 리드, DevEx 담당자
- CI/CD, 보안 리뷰, 코드 리뷰 프로세스에 익숙한 팀
- 모노레포 또는 멀티 레포 환경에서 에이전트 자동화 표준을 만들 책임이 있는 조직

## 교육 목적

1주 과정의 목적은 일회성 자동화가 아니라 사내에서 반복 사용 가능한 하네스 표준을 세우는 것입니다. 수강자는 스킬을 도메인 모듈처럼 설계하고, Task 그래프와 휴먼 승인 게이트를 포함한 엔드투엔드 흐름을 구현하며, 정적 분석·테스트·에이전트 스모크·운영 런북까지 연결합니다.

## 핵심 커리큘럼

| 일차 | 주제 | 핵심 활동 |
| --- | --- | --- |
| 월요일 | 표준과 레포 구조 | 기존 스택 매핑, 레포 레이아웃, 브랜치 전략 |
| 화요일 | 스킬 생명주기 | 버전 정책, 브레이킹 변경, 부분 실패 복구 |
| 수요일 | Task·서브에이전트 아키텍처 | 바운디드 컨텍스트, 병렬·파이프라인·휴먼인더루프 |
| 목요일 | `AGENTS.md`·거버넌스·보안 | PR 체크리스트, 시크릿·PII·프롬프트 인젝션 대응 |
| 금요일 | 검증·운영·로드맵 | CI 잡, 회귀 픽스처, 런북, 다음 분기 확장 계획 |

## 얻을 수 있는 결과

- 참조 구현 레포 또는 브랜치: 스킬 세트, 오케스트레이터, 선택적 서브에이전트 스텁
- 경량 아키텍처 결정 기록: Task 분해, 동시성, 휴먼 게이트 결정
- `AGENTS.md`와 신규 스킬 기여 가이드
- CI 설정 초안 또는 로컬 검증 스크립트와 회귀 fixture
- 보안 체크리스트: 데이터 분류, 로그 마스킹, 시크릿 항목
- 장애 확인 순서, 롤백, 연락망이 포함된 런북
- 다음 스프린트에 확장할 스킬·도구 모듈 로드맵

## 추천 활용 상황

- 회사 또는 플랫폼 조직 차원의 에이전트 하네스 표준을 만들 때
- 파일럿 이후 운영, 검증, 보안, 롤백까지 책임져야 할 때
- 비개발 1주 과정과 병행해 요구사항 완결성과 기술 구현 표준을 동시에 맞추고 싶을 때

## 준비물

스테이징 환경 또는 모킹 계정, 보안 가이드라인, 관측 도구 읽기 권한, CI/CD 후보 경로, 레포 구조 자료, 모델 게이트웨이 또는 런타임 제약 정보를 준비합니다.
