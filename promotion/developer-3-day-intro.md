# AX 워크톤 스프린트(3일) — 개발자 트랙 소개

## 한 줄 소개

**AX 워크톤 스프린트** 개발자 트랙은 팀 표준에 맞춘 스킬 패키지와 오케스트레이터를 설계하고, Task·서브에이전트·검증 전략까지 파일럿 수준으로 조립하는 3일 심화 스프린트입니다.

## 누구에게 맞나요

- 서비스, 플랫폼, 내부 도구를 유지보수하는 중급 이상 개발자
- 코드 리뷰와 Git 브랜치 전략에 익숙한 팀
- 에이전트 자동화를 개인 실험이 아니라 팀 표준 워크플로로 운영하고 싶은 조직

## 교육 목적

3일 과정은 단일 스킬 작성에서 한 단계 나아가, 스킬 간 의존성, 오케스트레이터의 상태 전달, 서브에이전트 병렬 실행, 실패 모드와 재시도 정책을 다룹니다. 마지막에는 `AGENTS.md`, 보안 주의, 에이전트 스모크 검증, 롤아웃 체크리스트까지 연결해 파일럿 도입에 필요한 기술 산출물을 만듭니다.

## 핵심 커리큘럼

| 일차 | 주제 | 핵심 활동 |
| --- | --- | --- |
| 1일차 | 스킬 패키지와 계약 | 역할 분리, `SKILL.md` 버전닝, 오케스트레이터 상태 전달 |
| 2일차 | Task·서브에이전트 심화 | 병렬 실행, 멱등성, 중복 억제, 재귀 방지 |
| 3일차 | `AGENTS.md`·검증·운영 | 모노레포 스코프, 시크릿 주의, 스모크 테스트, 롤아웃 발표 |

## 얻을 수 있는 결과

- 최소 2개 `SKILL.md`와 오케스트레이터 1개로 구성된 스킬 패키지
- Task 설계 문서: 서브에이전트 목록, 실패 모드, 재시도 정책
- `AGENTS.md` 업데이트: 포인터, 금지 경로, 로컬 실행 명령
- 단위 또는 스모크 스크립트 1개 이상과 실행법
- 모니터링 항목과 롤백 조건이 포함된 롤아웃 체크리스트

## 추천 활용 상황

- 개발팀 내부에서 에이전트 자동화 파일럿을 실제 브랜치로 추진할 때
- 여러 스킬을 조합하는 워크플로에서 실패 처리와 검증 기준이 필요한 때
- 비개발 트랙 산출물을 입력 fixture로 받아 기술 구현으로 연결할 때

## 준비물

개발자 1일 과정 준비물에 더해 테스트 러너, 샘플 API 모킹 수단, CI 후보 경로, 로그 검색 또는 관측 도구 접근 권한이 있으면 좋습니다.
