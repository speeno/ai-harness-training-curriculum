# 개발자 트랙 · 1일 과정

**형태:** 핸즈온 랩 중심(약 7시간, 점심 포함 시 8시간)

---

## 1. 교육 목표 (회사 니즈: 업무 자동화·생산성)

- Cursor 하네스에서 **에이전트 정의**, **SKILL.md**, **오케스트레이터 스킬**의 역할을 구분해 설명할 수 있다.
- **Task·서브에이전트**를 언제 쓰는지 판단하고, 최소 예제로 호출 흐름을 따라가 본다.
- **`AGENTS.md` 포인터**로 저장소 진입점을 문서화하는 패턴을 적용한다.
- **검증 개념**(스모크·회귀·로그 기준)을 에이전트 실행에 연결한다.

---

## 2. 대상 수강자

- 소프트웨어 엔지니어, 플랫폼·DevEx, 데이터·자동화 엔지니어 등 **리포지토리와 스크립트**에 익숙한 인원
- 사내 Git 및 IDE 사용이 가능한 환경

## 4. 강의 내용 상세 (모듈/토픽)

- **에이전트 정의:** 목적, 도구 범위, 금지 작업, 출력 형식
- **SKILL.md:** YAML 프런트매터(해당 시), 단계별 지침, 실패 시 행동
- **오케스트레이터 스킬:** 서브태스크 순서, 컨텍스트 전달 최소화, 중복 호출 방지
- **Task / 서브에이전트:** I/O 계약, 타임아웃·idempotency 고려, 무한 위임 방지
- **AGENTS.md:** 진입 스킬, 빌드·테스트 명령 포인터, 보안 주의
- **검증:** golden path 한 건, 회귀 체크리스트, 에이전트 로그에서 성공 판별 기준

---

## 5. 진행 방법 (강사 진행 팁, 실습 순서, 소그룹/페어, 준비물)

- **강사 팁:** 오전은 문법보다 **계약(입출력·경계)**를 강조한다. Task 예제는 한 번에 하나의 실패 모드만 넣는다.
- **실습 순서:** (1) 빈 레포 또는 연습 브랜치 생성 → (2) SKILL.md 초안 → (3) 오케스트레이터에서 서브태스크 호출 스텁 → (4) AGENTS.md에 포인터 추가 → (5) 스모크 스크립트 또는 체크리스트 실행.
- **페어:** Task 분해 페어 프로그래밍(한 명은 분해, 한 명은 리스크 목록).
- **준비물:** 노트북, Cursor, Git, 사내 샘플 레포 클론 권한, Node 또는 Python 등 팀 표준 런타임(선택).

---

## 6. 하네스·스킬 연계 (개발자 관점)

| 요소 | 1일 과정에서 다루는 범위 |
|------|---------------------------|
| **에이전트 정의** | 단일 목표 에이전트 1개 + 스킬 연결 |
| **SKILL.md** | 재사용 가능한 지침 블록, 버전 주석 |
| **오케스트레이터 스킬** | 하위 스킬 호출 순서만 명시(구현 디테일은 다음 과정) |
| **Task·서브에이전트** | 동기식 예제 1건, 재귀 금지 패턴 설명 |
| **AGENTS.md** | 스킬 인덱스·명령 포인터·금지 경로 |
| **검증** | 스모크 1건 + 로그에서 확인할 필드 목록 |

비개발 트랙 산출물(업무 시나리오)이 있으면 랩 입력으로 사용하면 현장감이 높아진다.

---

## 7. 산출물 (수강생 결과물)

1. **연습 레포 또는 브랜치:** SKILL.md 1개 이상, 오케스트레이터 스킬 초안 1개.
2. **AGENTS.md:** 최소 한 개 포인터(스킬 또는 빌드 명령).
3. **Task 분해 노트:** 서브에이전트 경계와 금지 사항 5줄 이내.
4. **검증 메모:** 스모크 절차와 기대 로그 키워드.
