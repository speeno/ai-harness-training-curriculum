# 개발자 트랙 · 1주 과정

**형태:** 5영업일(하루 6~7시간 + 일일 과제). 분산 일정 시 **총 랩 시간이 5일 분**이 되도록 조정한다.

---

## 1. 교육 목표 (회사 니즈: 업무 자동화·생산성)

- 사내 **하네스 표준**(에이전트·스킬·오케스트레이션·검증)을 코드와 문서로 동시에 확립한다.
- **모노레포/멀티 레포** 환경에서 스킬 재사용·버전 호환 정책을 설계한다.
- **Task·서브에이전트·사람 승인 게이트**(해당 시)를 포함한 엔드투엔드 흐름을 구현한다.
- **검증 파이프라인**(정적 분석·테스트·에이전트 스모크·선택적 E2E)을 제안하고 일부를 실제 연동한다.
- 운영 관점에서 **로그·알림·롤백**을 포함한 런북 초안을 작성한다.

---

## 2. 대상 수강자

- 아키텍트·시니어 엔지니어·플랫폼 리드 등 **표준 제정·리뷰** 역할이 가능한 인원
- CI/CD·보안 리뷰 프로세스와 협업 경험
- (선택) 비개발 트랙 1주와 **동일 시기**에 페어로 진행하면 요구-구현 정렬이 빨라진다.

---

## 3. 커리큘럼 표 (일차별)

### 월요일 — 표준과 레포 구조

| 블록 | 내용 |
|------|------|
| 오전 | 하네스 참조 모델, 팀 기존 스택과의 매핑 |
| 오후 | 레포 레이아웃, SKILL 네이밍, 브랜치 전략 |

### 화요일 — 스킬 생명주기

| 블록 | 내용 |
|------|------|
| 오전 | 스킬 버전 시맨틱, 브레이킹 변경 프로세스 |
| 오후 | 오케스트레이터 스킬 고급 패턴, 부분 실패 복구 |

### 수요일 — Task·서브에이전트 아키텍처

| 블록 | 내용 |
|------|------|
| 오전 | 바운디드 컨텍스트별 에이전트, 컨텍스트 전달 최적화 |
| 오후 | 병렬·파이프라인·휴먼인더루프 설계 랩 |

### 목요일 — AGENTS.md·거버넌스·보안

| 블록 | 내용 |
|------|------|
| 오전 | AGENTS.md·규칙·템플릿 PR 체크리스트 |
| 오후 | 시크릿·PII·프롬프트 인젝션 대응, 감사 로그 |

### 금요일 — 검증·운영·로드맵

| 블록 | 내용 |
|------|------|
| 오전 | 테스트 피라미드·픽스처·샘플 회귀 |
| 오후 | 런북·SLO 초안, 다음 분기 확장 로드맵 발표 |

---

## 4. 강의 내용 상세 (모듈/토픽)

- **아키텍처:** 스킬을 도메인 모듈처럼 다루기, 순환 의존 탐지
- **오케스트레이터:** 사가 패턴에 준하는 보상, 사용자 노티피케이션 템플릿
- **Task:** 스키마 검증 라이브러리 선택, 실패 타입 계층
- **서브에이전트:** 역할 스코프·토큰 예산·도구 화이트리스트
- **AGENTS.md:** 모노레포 루트 vs 패키지별 포인터, 중복 제거 규칙
- **검증:** 결정성 경계 식별, 비결정적 단계의 샘플 크기·플레이크 대응
- **보안·컴플라이언스:** 데이터 분류, 로그 마스킹, 승인 로그 보존
- **운영:** 대시보드 지표, 에러 예산, 피처 플래그와 에이전트 롤아웃

---

## 5. 진행 방법 (강사 진행 팁, 실습 순서, 소그룹/페어, 준비물)

- **강사 팁:** 수·목요일에 **크로스 리뷰**(보안 담당 30분 참관) 슬롯을 넣으면 현실성이 오른다.
- **실습 순서:** 월요일에 기준 레포 고정 → 매일 그 레포에 PR 형태로 누적 → 금요일에 태그 릴리스 시뮬레이션.
- **소그룹:** 스모크 테스트 설계는 **테스트 스웜**(4인, 각자 한 실패 모드 담당).
- **준비물:** developer 3일 준비물 + 스테이징 환경 또는 모킹 계정, 보안 가이드라인 링크, 관측 도구 읽기 권한.

---

## 6. 하네스·스킬 연계 (개발자 관점)

| 일차 | 기술 초점 | 산출 연결 |
|------|-----------|-----------|
| 월 | 레포·모듈 경계 | 팀 표준 폴더 스캐폴드 |
| 화 | 스킬 버전·호환 | CHANGELOG·마이그레이션 노트 |
| 수 | Task 그래프 | 시퀀스/병렬 혼합 구현 |
| 목 | AGENTS.md·보안 | PR 템플릿·리뷰 룰 |
| 금 | 검증·운영 | CI 잡·런북·SLO 초안 |

비개발 트랙 산출물이 있으면 **fixture·인수 조건**으로 직접 연결한다.

---

## 7. 산출물 (수강생 결과물)

1. **참조 구현 레포(또는 브랜치):** 스킬 세트 + 오케스트레이터 + 선택적 서브에이전트 스텁.
2. **아키텍처 결정 기록(경량):** Task 분해·동시성·휴먼 게이트 결정 2~3건.
3. **AGENTS.md·기여 가이드:** 신규 스킬 추가 절차.
4. **검증:** CI 설정 초안 또는 로컬 스크립트 + 회귀 픽스처.
5. **보안 체크리스트:** 데이터 분류·로그·시크릿 항목.
6. **런북:** 장애 시 확인 순서, 롤백, 연락망.
7. **로드맵:** 다음 스프린트에 확장할 스킬·도구 모듈 목록.

---

## 8. 선택 도구 모듈 (회사별 도구 미정 시 플레이스홀더)

1주 과정에서는 **금요일 오전**까지 아래 표를 채우는 것을 완료 조건에 포함한다.

| 레이어 | 플레이스홀더 | 결정 시 기록할 내용 |
|--------|----------------|---------------------|
| 소스 | `[Git 호스팅]` | 브랜치 보호 규칙 |
| CI | `[파이프라인 파일]` | 스모크 잡 ID |
| 아티팩트 | `[레지스트리]` | 스킬 패키지 배포 방식 |
| 런타임 | `[모델 게이트웨이]` | 엔드포인트·쿼터 |
| 관측 | `[APM/로그]` | 대시보드 링크 |
| 시크릿 | `[주입 메커니즘]` | 변수 이름 규약 |

미정 항목은 **RFC 한 페이지**로 남기고, 소유자·목표 날짜만표시한다.
