# 비개발자 트랙 · 1주 과정

**형태:** 5영업일 기준(하루 6시간 핵심 + 일일 과제 30~60분). 주차 중 부서 일정에 맞춰 쉬는 날를 넣을 경우 **총 학습량은 5일 분**으로 환산한다.

---

## 1. 교육 목표 (회사 니즈: 업무 자동화·생산성)

- 부서·크로스펑션 팀이 **동일한 언어(스킬·워크플로·워크스페이스)**로 자동화 로드맵을 논의할 수 있다.
- 스킬을 **버전 관리되는 업무 자산**으로 정의하고, 분기별 개정 리듬을 설계한다.
- **파일럿 1건**을 끝까지 설계하고, **현업 검증(샘플 데이터)**까지 포함한 패키지를 완성한다.
- 선택적으로 **경영·IT 스폰서**에게 보고할 수 있는 브리프를 작성한다.

---

## 2. 대상 수강자

- **변경 관리·품질·컴플라이언스**에 관심 있는 시니어 비개발 직군
- 여러 팀을 아우르는 **PMO·운영 총괄** 등 조정 역할
- 사내 AI 거버넌스 워킹그룹 예비 멤버

---

## 3. 커리큘럼 표 (일차별)

### 월요일 — 전략 정렬

| 시간 | 내용 |
|------|------|
| 오전 | 회사 생산성 목표·제약(데이터·보안) 정렬 |
| 오후 | 자동화 포트폴리오 초안, 스킬 개념 합의 |

### 화요일 — 스킬 공학(비기술)

| 시간 | 내용 |
|------|------|
| 오전 | 스킬 목차 표준, 용어집, 금지 행동 |
| 오후 | 팀 템플릿 작성, 페어 교차 리뷰 2라운드 |

### 수요일 — 워크플로 심화

| 시간 | 내용 |
|------|------|
| 오전 | 오케스트레이션 패턴(분기·병렬·승인) |
| 오후 | 시퀀스·스윔레인 수준 다이어그램, 예외 경로 |

### 목요일 — 워크스페이스와 감사 가능성

| 시간 | 내용 |
|------|------|
| 오전 | `_workspace` 트리, 보존 기간, 접근 권한 |
| 오후 | 샘플 데이터로 "폴더만 따라가도 재현되는지" 리허설 |

### 금요일 — 파일럿 패키지·브리프

| 시간 | 내용 |
|------|------|
| 오전 | KPI·성공·중단 조건, 파일럿 체크리스트 |
| 오후 | 발표 리허설, 스폰서 Q&A 시뮬레이션 |

---

## 4. 강의 내용 상세 (모듈/토픽)

- **전략:** 빠른 실패보다 **감사 가능한 최소 파일럿**을 선택하는 기준
- **스킬 심화:** 버전 번호 규칙, 변경 로그, "이 스킬을 빼면 무엇이 불가능한지" 한 줄 설명
- **워크플로:** SLA, 에스컬레이션, 고객 대면 여부에 따른 게이트
- **`_workspace`:** 개인 하드코딘이 아닌 **팀 공유 드라이브 경로**와 매핑
- **검증(비개발):** 체크리스트, 샘플 입력→기대 출력 표, 오류 유형 분류
- **커뮤니케이션:** IT·법무·보안과의 질문 리스트, 브리프 1페이지 구조

---

## 5. 진행 방법 (강사 진행 팁, 실습 순서, 소그룹/페어, 준비물)

- **강사 팁:** 매일 시작 15분은 **전일 과제 리캡**으로 고정하고, 금요일 오전에만 경영 관점 프레이밍을 추가한다.
- **실습 순서:** 월·화에 템플릿 고정 → 수·목에 시나리오에 맞게 채움 → 금요일에 패키징.
- **소그룹:** **페어 프로그래밍처럼** "스킬 페어": 한 명은 작성, 한 명은 질문만 하는 역할 교대.
- **준비물:** 노트북, IDE, 비식별 샘플 데이터 세트(가능 시 3종: 정상·경계·오류), 조직도, 관련 규정 요약(1~2페이지).

---

## 6. 하네스·스킬 연계 (비개발자 관점)

| 주차 일차 | 초점 |
|-----------|------|
| 월~화 | 스킬을 **조직 자산**으로 규정, 개인 프롬프트와 분리 |
| 수 | 오케스트레이터를 **정책 반영 장치**로 설명(예: 고액만 이중 승인) |
| 목 | `_workspace`를 **추적 가능한 작업 공간**으로 연결 |
| 금 | 산출물을 개발 트랙·거버넌스 보드에 **넘기는 인터페이스** 정의 |

기술 스택을 모르더라도 **요구사항 완결성**을 검증하는 것이 본 과정의 핵심 산출이다.

---

## 7. 산출물 (수강생 결과물)

1. **자동화 포트폴리오 표:** 후보 5건 이상, 점수화 근거.
2. **스킬 v1.x:** 목차·용어집·본문·예외·부록(양식 링크).
3. **워크플로 패키지:** 다이어그램 + 역할 책임표(RACI 간단版).
4. **`_workspace` 운영 가이드:** 트리, 명명, 보존, 접근.
5. **검증 체크리스트:** 최소 10항목, 통과/실패 예시.
6. **스폰서 브리프(선택):** 1페이지, 요청·기대효과·리스크·다음 단계.

---

## 8. 선택 도구 모듈 (회사별 도구 미정 시 플레이스홀더)

1주 과정에서는 도구를 **주차 중 하나로 확정**하는 것을 목표로 한다.

| 영역 | 플레이스홀더 | 확정 시 채울 정보 |
|------|----------------|-------------------|
| 지식 저장소 | `[위키/Notion/Confluence]` | 스킬 게시 위치, 읽기 권한 |
| 과제·승인 | `[과제 시스템]` | 파일럿 티켓 템플릿 ID |
| 커뮤니케이션 | `[채팅/메일]` | 에스컬레이션 채널 |
| 파일 | `[클라우드 스토리지]` | `_workspace` 실제 루트 경로 |
| 모델 접근 | `[사내 게이트웨이]` | 허용 모델·금지 데이터 |

미확정 상태로 종료 시 **"도구 매핑 표"**만이라도 남기고, 확정 후 2주 이내 후속 워크숍 일정을 권장한다.
