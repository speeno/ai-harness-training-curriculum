# 03. 비개발자 트랙 · 1일 과정 실습 워크북

> 과정명: 하네스(Harness) · Agent Skills 기반 기업 AI 교육  
> 트랙: 비개발자 1일 집중 워크숍  
> 목적: 반복 업무 1건을 **업무 시나리오 + Skill 초안 + Workflow + Workspace + 검증 체크리스트**로 정리한다.

---

# 0. 워크북 사용 방법

이 워크북은 강의 중 직접 작성하는 실습용 자료입니다.  
완벽한 문서를 만드는 것이 아니라, 우리 팀이 함께 논의할 수 있는 **업무 자동화 초안**을 만드는 것이 목표입니다.

오늘 작성할 최종 산출물은 다음과 같습니다.

- 업무 후보 선정표
- 업무 시나리오 1페이지
- Skill 초안
- 승인·검토 Workflow
- `_workspace` 폴더 규약
- 리스크 레지스터
- 검증 체크리스트
- 다음 액션 플랜

## 4.2 목적 작성

이 Skill이 무엇을 돕기 위한 것인지 한두 문장으로 작성합니다.

| 질문 | 작성 내용 |
| --- | --- |
| 이 Skill은 어떤 업무를 돕는가? |  |
| 이 Skill을 사용하면 누가 편해지는가? |  |
| 이 Skill의 결과물은 어떤 의사결정이나 후속 업무에 쓰이는가? |  |

목적 문장 초안:

```text
이 Skill은 __________________ 업무에서 __________________ 입력 자료를 바탕으로
__________________ 결과물을 작성하여 __________________ 사람이 빠르게 검토하고 활용할 수 있도록 돕는다.
```

작성 예시:

```text
이 Skill은 주간 회의록 정리 업무에서 회의 녹취록과 참석자 메모를 바탕으로
핵심 논의사항, 결정사항, 액션 아이템을 작성하여 팀장이 빠르게 검토하고 공유할 수 있도록 돕는다.
```

# 6. 실습 5 — `_workspace` 폴더 규약 작성

## 6.1 기본 폴더 구조

아래 기본 구조를 우리 팀 업무에 맞게 수정합니다.

```text
_workspace/
  in/        원본 입력 자료
  work/      작업 중 파일, 중간 결과
  out/       최종 산출물
  archive/   완료 후 보관 자료
  tmp/       임시 파일
```

## 14.2 예시 Skill 초안

```markdown
# Skill 초안: 주간 회의록 요약 및 액션 아이템 정리

## 1. 업무명
주간 회의록 요약 및 액션 아이템 정리

## 2. 목적
회의록 원문을 바탕으로 핵심 논의사항, 결정사항, 액션 아이템을 정리하여 팀장이 빠르게 검토하고 팀원에게 공유할 수 있도록 돕는다.

## 3. 적용 범위
정기 주간 회의, 프로젝트 회의, 내부 운영 회의에 사용한다. 외부 고객과의 민감한 계약 협상 회의에는 별도 승인 후 사용한다.

## 4. 입력 자료
- 회의록 원문
- 참석자 목록
- 프로젝트명
- 회의 일자

## 5. 처리 절차
1. 입력 자료가 모두 있는지 확인한다.
2. 회의의 핵심 논의사항을 3~5개로 요약한다.
3. 결정사항을 별도 항목으로 정리한다.
4. 액션 아이템을 담당자, 기한, 작업 내용으로 구분한다.
5. 확인이 필요한 불명확한 내용을 별도 표시한다.

## 6. 금지 사항 / 주의 사항
- 회의록에 없는 내용을 추정해서 추가하지 않는다.
- 담당자가 불명확한 경우 임의로 지정하지 않는다.
- 민감한 개인정보는 최종 공유본에 포함하지 않는다.

## 7. 예외 처리
- 회의록이 불완전하면 “입력 자료 부족”으로 표시한다.
- 담당자나 기한이 불명확하면 “확인 필요”로 표시한다.

## 8. 출력 형식
- 핵심 요약
- 결정사항
- 액션 아이템 표
- 확인 필요 사항

## 9. 사람 검토 지점
팀장이 최종 공유 전에 담당자, 기한, 민감정보 포함 여부를 확인한다.

## 10. 완료 기준
팀장 검토가 완료되고 최종본이 `_workspace/out`에 저장되면 완료로 본다.
```
