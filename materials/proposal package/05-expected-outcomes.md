# 교육 후 기대 효과

## 1. 조직 관점 기대 효과

| 기대 효과 | 설명 |
| --- | --- |
| AI 활용 표준화 | 개인별 프롬프트 사용을 팀 표준 Skill로 전환 |
| 업무 요구사항 정렬 | 현업 요구를 개발자가 이해 가능한 산출물로 정리 |
| 승인·검토 체계화 | 사람 검토 지점과 책임 구조 명확화 |
| 산출물 관리 개선 | `_workspace` 규칙으로 원본·초안·최종본 구분 |
| 검증 가능성 확보 | 체크리스트, Smoke Test, Regression 기준 마련 |
| 보안 리스크 감소 | 민감정보, 시크릿, 로그, 외부 전송 기준 포함 |
| 파일럿 전환 가속 | 교육 결과물이 바로 파일럿 검토 자료로 활용 가능 |
| 확장 로드맵 도출 | 다음 스프린트 또는 다음 분기 과제 정리 |

---

## 2. 비개발자 관점 기대 효과

비개발자 수강자는 교육 후 다음을 수행할 수 있습니다.

- 반복 업무를 자동화 가능한 요구사항으로 정리할 수 있다.
- AI가 할 일과 사람이 검토할 일을 구분할 수 있다.
- Skill을 업무 매뉴얼로 작성할 수 있다.
- 승인·예외·재작업 Workflow를 설계할 수 있다.
- 산출물 위치와 파일명 규칙을 정의할 수 있다.
- 검증 체크리스트와 리스크 레지스터를 만들 수 있다.
- 개발자 트랙 또는 파일럿으로 넘길 업무 패키지를 작성할 수 있다.

---

## 3. 개발자 관점 기대 효과

개발자 수강자는 교육 후 다음을 수행할 수 있습니다.

- 현업 Skill 초안을 `SKILL.md`로 전환할 수 있다.
- Orchestrator와 Task/Sub-agent 구조를 설계할 수 있다.
- `AGENTS.md`로 저장소 진입점과 작업 규칙을 제공할 수 있다.
- Smoke Verification과 Failure Matrix를 작성할 수 있다.
- 보안·로그·원본 보호 기준을 하네스 구조에 반영할 수 있다.
- 파일럿 브랜치 또는 참조 구현으로 확장할 수 있다.
- CI 또는 로컬 검증 파이프라인으로 확장할 수 있다.

| 관점 | 기대 효과 |

## 5. 교육 산출물의 후속 활용

| 산출물 | 후속 활용 |
| --- | --- |
| 업무 시나리오 | 파일럿 범위 정의, 개발자 요구사항 입력 |
| Skill 초안 | 팀 표준 업무 매뉴얼, `SKILL.md` 작성 기반 |
| Workflow | 승인·검토 절차, Orchestrator 설계 기반 |
| Workspace 규칙 | 산출물 관리, 감사 가능성 확보 |
| 검증 체크리스트 | Smoke Test, Regression fixture 기반 |
| Risk Register | Failure Matrix, 중단 조건, 보안 검토 기반 |
| `AGENTS.md` | 저장소 진입점, 작업 규칙, 금지 경로 안내 |
| Runbook | 파일럿 운영과 장애 대응 기준 |
| Roadmap | 다음 스프린트 또는 다음 분기 확장 계획 |

---

## 6. 파일럿 전환 효과

3일 이상 과정의 산출물은 실제 파일럿 검토 자료로 활용할 수 있습니다.

파일럿 전환 시 기대 효과:

- 파일럿 대상 업무 범위가 명확해짐
- 입력 데이터와 기대 출력이 정의됨
- 사람 검토와 승인 지점이 명확해짐
- 성공 기준과 중단 조건이 사전에 정리됨
- 개발자 구현 범위와 검증 기준이 연결됨
- 보안 검토와 운영 준비가 쉬워짐

---

## 7. 장기 확산 효과

1주 과정까지 진행한 경우 다음과 같은 장기 효과를 기대할 수 있습니다.

- 부서별 Skill 표준화
- 자동화 포트폴리오 관리
- 조직 차원의 AI Workflow 거버넌스 수립
- 검증 fixture와 Smoke Test 축적
- 보안·로그·승인 기준의 재사용
- 플랫폼 조직의 Agent Harness 표준 확립
- 사내 AI 자동화 로드맵 수립
