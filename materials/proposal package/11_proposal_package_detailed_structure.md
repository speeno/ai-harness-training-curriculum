# 1. `00-one-page-summary.md`

## 1.1 파일 목적

전체 교육 프로그램을 한 페이지로 요약하는 문서입니다.  
경영진, 부서장, 교육 승인권자, 고객사 담당자가 가장 먼저 읽는 자료입니다.

## 1.2 권장 분량

- 1페이지
- A4 기준 700~1,000자 내외
- 표 2~3개 포함 가능

## 1.3 포함할 내용

```markdown
# 하네스 · Agent Skills 기반 기업 AI 교육 요약

## 1. 교육 목적

## 2. 왜 필요한가

## 3. 교육 트랙

## 4. 기간별 과정

## 5. 교육 후 산출물

## 6. 기대 효과

## 7. 추천 도입 방식
```

---

## 1.4 작성 예시

```markdown
# 하네스 · Agent Skills 기반 기업 AI 교육 요약

## 1. 교육 목적

본 교육은 AI 활용을 개인별 프롬프트 중심에서 조직이 재사용할 수 있는 Skill, Workflow, Workspace, 검증 체계로 전환하기 위한 기업 실무 교육입니다.

## 2. 왜 필요한가

많은 조직이 AI 도구를 도입했지만, 실제 현장에서는 개인별 사용 방식, 산출물 위치 혼선, 승인 누락, 검증 부재로 인해 성과가 반복되지 않는 문제가 발생합니다. 본 교육은 이러한 문제를 업무 표준화와 개발 하네스 구축 관점에서 해결합니다.

## 3. 교육 트랙

| 트랙 | 대상 | 핵심 산출물 |
| --- | --- | --- |
| 비개발자 | 기획, 운영, PM, 행정, 영업 지원, 품질, 컴플라이언스 | 업무 시나리오, Skill 초안, Workflow, Workspace 규칙, 검증 체크리스트 |
| 개발자 | 개발자, 플랫폼, DevEx, 데이터·자동화 엔지니어 | `SKILL.md`, Orchestrator, Task, `AGENTS.md`, Smoke Verification, Security Notes |

## 4. 기간별 과정

| 과정 | 추천 상황 | 결과물 |
| --- | --- | --- |
| 1일 | 도입 워크숍 | 최소 자동화 초안 |
| 3일 | 팀 파일럿 준비 | 파일럿 설계 패키지 |
| 1주 | 조직 표준화 | 거버넌스·운영·로드맵 패키지 |

## 5. 교육 후 기대 효과

- 개인별 프롬프트를 팀 표준 Skill로 전환
- 현업 요구사항과 개발 구현 간 전달 손실 감소
- 승인, 검토, 보안, 검증 기준 명확화
- 파일럿 실행과 다음 스프린트 과제 도출

## 6. 추천 도입 방식

AI 도입 초기 조직은 비개발자 1일 + 개발자 1일 과정으로 공통 언어를 정렬하고, 특정 파일럿을 준비하는 조직은 비개발자 3일 + 개발자 3일 과정을 권장합니다. 전사 표준과 거버넌스를 준비하는 조직은 1주 과정을 권장합니다.
```

## 2.4 섹션별 작성 내용

### 1. 프로그램 개요

작성 방향:

- 이 교육이 일반적인 생성형 AI 사용법 교육이 아님을 명확히 함
- 조직의 반복 가능한 AI 자동화 운영 체계 구축이 목적임을 설명

예시 문구:

```markdown
하네스(Harness) · Agent Skills 기반 기업 AI 교육은 사내 AI 활용을 개인별 프롬프트 역량에만 의존하지 않고, 팀과 조직이 함께 관리할 수 있는 업무 절차와 개발 표준으로 정착시키는 교육 프로그램입니다.
```

### 2. 문제의식

포함할 표:

| 문제 | 현상 | 결과 |
| --- | --- | --- |
| 개인별 프롬프트 의존 | 사람마다 요청 방식이 다름 | 품질 편차 발생 |
| 산출물 위치 혼선 | 원본·초안·최종본이 섞임 | 재현과 감사 어려움 |
| 승인 누락 | AI 결과가 검토 없이 사용됨 | 업무 리스크 증가 |
| 검증 부재 | 맞는 결과인지 판단 기준이 없음 | 파일럿 확산 어려움 |
| 개발 연결 단절 | 현업 요구가 구현 기준으로 전달되지 않음 | 자동화 지연 |

### 3. Prompt에서 Skill로

핵심 문구:

```markdown
프롬프트는 개인의 요청문입니다. Skill은 조직이 재사용할 수 있는 업무 절차입니다.
```

비교표:

| Prompt 중심 | Skill 중심 |
| --- | --- |
| 개인이 기억하는 요청 방식 | 조직이 관리하는 업무 매뉴얼 |
| 매번 달라지는 표현 | 표준화된 입력·출력 기준 |
| 검토 기준 불명확 | 사람 승인과 검증 기준 포함 |
| 재사용 어려움 | 버전 관리와 개선 가능 |

### 4. 하네스란 무엇인가

도식:

```text
Business Scenario
   ↓
Agent
   ↓
Skill / SKILL.md
   ↓
Orchestrator / Task / Sub-agent
   ↓
Workspace
   ↓
Verification / Human Review
   ↓
Approved Output
```

### 5. Agent Skills란 무엇인가

설명:

```markdown
Agent Skill은 에이전트가 특정 업무를 수행할 때 참조하는 재사용 가능한 지침 단위입니다. 비개발자에게는 업무 매뉴얼이며, 개발자에게는 `SKILL.md`로 관리되는 업무 계약입니다.
```

## 3.3 핵심 비교표

```markdown
| 구분 | 비개발자 트랙 | 개발자 트랙 |
| --- | --- | --- |
| 핵심 질문 | 어떤 업무를 어떤 절차와 규칙으로 자동화할 것인가? | 에이전트가 안전하게 도구를 쓰고 Skill을 재사용·검증하려면 무엇을 정의해야 하는가? |
| 주요 대상 | 기획, 운영, PM, 행정, 영업 지원, 품질, 컴플라이언스 | 개발자, 플랫폼, DevEx, 데이터·자동화 엔지니어 |
| Skill 의미 | 업무 매뉴얼, SOP, 금지 사항 | `SKILL.md`, 입력·출력 계약, 실패 처리 |
| Workflow 의미 | 승인·검토가 포함된 업무 흐름 | Orchestrator, context passing, human gate |
| Workspace 의미 | 입력·초안·최종본을 나누는 폴더 규칙 | 파일 경로 계약, 원본 보호, audit trail |
| 검증 방식 | 체크리스트, 샘플 입력·기대 출력 | Smoke Test, Regression fixture, Log keys |
| 대표 산출물 | 업무 시나리오, Skill 초안, Workflow, Risk Register | `SKILL.md`, Orchestrator, Task Breakdown, `AGENTS.md` |
```

```markdown
| 전사 표준·거버넌스 준비 | 비개발자 1주 + 개발자 1주 | 운영 모델, 보안, 검증, 로드맵까지 포함 |
```

## 10.3 FAQ 예시

```markdown
## Q1. 기존 ChatGPT 교육과 무엇이 다른가요?

이 교육은 프롬프트 작성법 중심 교육이 아닙니다. 조직이 반복 사용할 수 있는 Skill, Workflow, Workspace, 검증 체계를 만드는 데 초점을 둡니다.

## Q2. 비개발자도 참여할 수 있나요?

네. 비개발자는 업무 절차와 승인·검토 기준을 Skill과 Workflow로 정리하는 핵심 역할을 합니다.

## Q3. 개발자 과정에서는 실제 코딩을 하나요?

1일 과정은 하네스 구조와 문서 계약 중심이며, 3일·1주 과정에서는 참조 구현, 검증, 운영 구조까지 확장할 수 있습니다.

## Q4. 교육 후 바로 파일럿이 가능한가요?

조직의 준비 수준에 따라 다릅니다. 3일 이상 과정은 파일럿 검토에 필요한 설계 패키지를 목표로 합니다.

## Q5. 어떤 준비물이 필요한가요?

파일럿 후보 업무, 비식별 샘플 데이터, 도구 환경, 보안 기준, 저장소 또는 문서 공간이 필요합니다.

## Q6. 민감정보가 포함된 업무도 다룰 수 있나요?

실제 민감정보는 사용하지 않는 것이 원칙입니다. 비식별 샘플 또는 가상 데이터를 사용해야 하며, 민감정보 처리 기준은 별도 검토가 필요합니다.

## Q7. 1일, 3일, 1주 중 무엇을 선택해야 하나요?

AI 도입 초기라면 1일, 특정 파일럿을 준비한다면 3일, 조직 표준과 운영 체계를 만들려면 1주 과정을 권장합니다.

## Q8. 교육 산출물은 어디에 활용되나요?

업무 시나리오, Skill, Workflow, 검증 체크리스트는 파일럿 검토 자료로 활용되고, 개발자 산출물은 구현 브랜치나 참조 하네스의 기반으로 활용됩니다.
```
