# 하네스 · Agent Skills 기반 기업 AI 교육 요약

## 1. 교육 목적

본 교육은 AI 활용을 개인별 프롬프트 중심에서 조직이 재사용할 수 있는 **Skill, Workflow, Workspace, 검증 체계**로 전환하기 위한 기업 실무 교육입니다.

일반적인 AI 도구 사용법 교육이 아니라, 조직이 AI 자동화를 안전하고 반복 가능하게 운영하기 위한 **업무 표준화와 개발 하네스 구축 교육**입니다.

---

## 2. 왜 필요한가

많은 조직이 AI 도구를 도입했지만, 실제 현장에서는 다음 문제가 반복됩니다.

| 문제 | 설명 | 결과 |
| --- | --- | --- |
| 개인별 프롬프트 의존 | 사람마다 요청 방식이 다름 | 결과 품질 편차 발생 |
| 산출물 위치 혼선 | 입력, 초안, 최종본이 섞임 | 재현과 감사 어려움 |
| 승인·검토 누락 | AI 결과가 검토 없이 사용됨 | 업무·보안 리스크 증가 |
| 검증 기준 부재 | 맞는 결과인지 판단 기준이 없음 | 파일럿 확산 어려움 |
| 현업·개발 연결 단절 | 업무 요구가 구현 기준으로 전달되지 않음 | 자동화 구현 지연 |

---

## 3. 교육 트랙

| 트랙 | 대상 | 핵심 산출물 |
| --- | --- | --- |
| 비개발자 트랙 | 기획, 운영, PM, 행정, 영업 지원, 품질, 컴플라이언스 | 업무 시나리오, Skill 초안, Workflow, Workspace 규칙, 검증 체크리스트 |
| 개발자 트랙 | 개발자, 플랫폼, DevEx, 데이터·자동화 엔지니어 | `SKILL.md`, Orchestrator, Task, `AGENTS.md`, Smoke Verification, Security Notes |

---

## 4. 기간별 과정

| 과정 | 추천 상황 | 결과물 |
| --- | --- | --- |
| 1일 과정 | 도입 워크숍, 공통 언어 정렬 | 최소 자동화 초안 |
| 3일 과정 | 팀 단위 파일럿 준비 | 파일럿 설계 패키지 |
| 1주 과정 | 조직 표준·거버넌스 준비 | 거버넌스·운영·로드맵 패키지 |

---

## 5. 교육 후 산출물

교육 종료 후에는 다음 산출물이 남습니다.

- 업무 시나리오 1페이지
- Skill 초안 또는 Skill v1.x
- Workflow / Orchestrator 설계
- `_workspace` 운영 규칙
- 리스크 레지스터
- 검증 체크리스트 또는 Smoke Verification
- `AGENTS.md`
- Security Notes
- Rollout Checklist
- Runbook 또는 로드맵

---

## 6. 기대 효과

- 개인별 프롬프트를 팀 표준 Skill로 전환
- 현업 요구사항과 개발 구현 간 전달 손실 감소
- 승인, 검토, 보안, 검증 기준 명확화
- 산출물 관리와 감사 가능성 개선
- AI 자동화 파일럿 실행과 다음 스프린트 과제 도출

---

## 7. 추천 도입 방식

| 조직 상황 | 추천 조합 |
| --- | --- |
| AI 도입 초기 | 비개발자 1일 + 개발자 1일 |
| 특정 부서 파일럿 준비 | 비개발자 3일 + 개발자 3일 |
| 현업 요구는 있으나 구현 표준이 없음 | 비개발자 3일 후 개발자 3일 |
| 플랫폼 조직이 먼저 표준을 만들고 싶음 | 개발자 1주 후 비개발자 1일/3일 확산 |
| 전사 표준·거버넌스 준비 | 비개발자 1주 + 개발자 1주 |
