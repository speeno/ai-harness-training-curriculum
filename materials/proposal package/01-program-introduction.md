# 프로그램 소개

## 1. 프로그램 개요

하네스(Harness) · Agent Skills 기반 기업 AI 교육은 사내 AI 활용을 개인별 프롬프트 역량에만 의존하지 않고, 팀과 조직이 함께 관리할 수 있는 업무 절차와 개발 표준으로 정착시키는 교육 프로그램입니다.

이 교육은 단순한 생성형 AI 사용법 교육이 아닙니다. 교육의 핵심은 조직이 AI 자동화를 반복 가능하고, 안전하고, 검증 가능하게 운영할 수 있도록 **Skill, Workflow, Workspace, Verification** 체계를 만드는 것입니다.

---

## 2. 문제의식: AI 도구 도입 이후의 운영 문제

AI 도구는 빠르게 확산되고 있지만, 기업 현장에서는 다음과 같은 문제가 반복됩니다.

| 문제 | 현상 | 결과 |
| --- | --- | --- |
| 개인별 프롬프트 의존 | 사람마다 다른 방식으로 AI에 요청 | 결과 품질 편차 발생 |
| 산출물 위치 혼선 | 원본, 초안, 최종본이 섞임 | 재현과 감사 어려움 |
| 승인·검토 누락 | AI 결과가 검토 없이 사용됨 | 운영·보안 리스크 증가 |
| 검증 기준 부재 | 결과가 맞는지 판단 기준 없음 | 파일럿 확산 어려움 |
| 개발 연결 단절 | 현업 요구가 구현 기준으로 전달되지 않음 | 자동화 구현 지연 |

---

## 3. 교육의 핵심 전환: Prompt에서 Skill로

프롬프트는 개인의 요청문입니다.  
Skill은 조직이 재사용할 수 있는 업무 절차입니다.

| Prompt 중심 | Skill 중심 |
| --- | --- |
| 개인이 기억하는 요청 방식 | 조직이 관리하는 업무 매뉴얼 |
| 매번 달라지는 표현 | 표준화된 입력·출력 기준 |
| 검토 기준 불명확 | 사람 승인과 검증 기준 포함 |
| 재사용 어려움 | 버전 관리와 개선 가능 |
| 개발 전달 어려움 | `SKILL.md`, Task, Test로 전환 가능 |

---

## 4. 하네스(Harness)란 무엇인가

하네스는 AI 에이전트가 업무 안에서 안전하게 움직이도록 잡아주는 실행 구조입니다.

```text
Business Scenario
   ↓
Agent
   ↓
Skill / SKILL.md
   ↓
Orchestrator / Task / Sub-agent
   ↓
Workspace
   ↓
Verification / Human Review
   ↓
Approved Output
```

하네스는 모델 자체보다 운영 구조에 초점을 둡니다.  
기업 업무에서는 입력, 출력, 승인, 검증, 보안, 로그, 저장 위치가 함께 정의되어야 실제 자동화가 가능합니다.

---

## 5. Agent Skills란 무엇인가

Agent Skill은 에이전트가 특정 업무를 수행할 때 참조하는 재사용 가능한 지침 단위입니다.

| 관점 | 의미 |
| --- | --- |
| 비개발자 관점 | 업무 매뉴얼, SOP, 금지 사항 |
| 개발자 관점 | `SKILL.md`, 트리거, 입력·출력 계약, 실패 처리 |
| 조직 관점 | 버전 관리되는 업무 자산 |

Skill에는 다음 내용이 포함될 수 있습니다.

- 업무 목적
- 적용 범위
- 입력 자료
- 단계별 절차
- 금지 사항
- 예외 처리
- 출력 형식
- 검증 기준
- 사람 검토 지점

---

## 6. 비개발자와 개발자의 역할

비개발자는 업무를 설명 가능한 요구사항으로 정리합니다.

- 업무 시나리오
- Skill 초안
- 승인·검토 Workflow
- Workspace 규칙
- 리스크와 검증 체크리스트

개발자는 이를 실행 가능하고 검증 가능한 하네스로 전환합니다.

- `SKILL.md`
- Orchestrator Skill
- Task / Sub-agent 설계
- `AGENTS.md`
- Smoke Verification
- Security Notes
- Rollout Checklist

---

## 7. 교육 방식

교육은 강의 중심이 아니라 산출물 중심 워크숍으로 운영됩니다.

| 방식 | 설명 |
| --- | --- |
| 개념 설명 | Skill, Workflow, Workspace, Verification 개념 정렬 |
| 실습 | 업무 시나리오, Skill, Workflow, 검증 기준 작성 |
| 페어 리뷰 | 다른 사람이 읽어도 이해되는지 점검 |
| 발표 | 파일럿 가능성과 보완점 확인 |
| 후속 액션 | 내부 검토, 개발자 트랙, 파일럿 브랜치로 연결 |

---

## 8. 교육 후 남는 것

교육 후에는 단순 수강 결과가 아니라 실제 파일럿 또는 내부 표준화 논의에 사용할 수 있는 산출물이 남습니다.

- 1일 과정: 최소 자동화 초안
- 3일 과정: 파일럿 설계 패키지
- 1주 과정: 조직 표준·거버넌스·운영 로드맵

---

## 9. 추천 대상 조직

이 교육은 다음 조직에 적합합니다.

- AI 도구를 도입했지만 결과 품질이 반복되지 않는 조직
- 현업 요구사항과 개발 구현 사이의 전달 손실이 큰 조직
- 승인·검토·보안·검증 기준이 필요한 조직
- 사내 AI 자동화 파일럿을 준비하는 조직
- 플랫폼 또는 DevEx 차원에서 에이전트 하네스 표준을 만들고 싶은 조직
