# 10. 홍보·제안서 자료 정리

> 과정명: 하네스(Harness) · Agent Skills 기반 기업 AI 교육  
> 목적: 교육 프로그램을 내부 의사결정자, 고객사, 교육 담당자, 경영·IT 스폰서에게 설명하기 위한 홍보·제안 자료로 재구성한다.

---

# 1. 자료 목적

이 문서는 지금까지 작성한 교육 자료를 외부 안내, 내부 승인, 고객 제안, 교육 홍보에 사용할 수 있도록 정리한 자료입니다.

활용 대상:

- 내부 교육 기획자
- 경영진 또는 부서장
- IT·DX·AI 전환 담당자
- 고객사 교육 제안 담당자
- 플랫폼·DevEx 조직
- PMO·운영 혁신 조직

핵심 메시지:

```text
이 교육은 AI 도구 사용법 교육이 아니라,
조직이 AI 자동화를 안전하고 반복 가능하게 운영하기 위한
업무 표준화와 개발 하네스 구축 교육입니다.
```

## 1.1 의사결정자별 핵심 메시지

제안서나 소개 미팅에서는 같은 교육이라도 상대의 관심사에 맞춰 강조점을 바꾼다.

| 대상 | 관심사 | 강조 메시지 | 보여줄 자료 |
| --- | --- | --- | --- |
| 경영진·부서장 | 생산성, 리스크, 확산 가능성 | 개인별 AI 사용을 조직 표준과 운영 모델로 전환한다. | 1페이지 요약, 기대 효과, 30/60/90일 로드맵 |
| 교육·HR 담당자 | 교육 효과, 수강생 산출물, 운영 난이도 | 강의가 아니라 실습 산출물이 남는 과정이다. | 과정별 시간표, 산출물 목록, 평가 체크리스트 |
| DX·AI 전환 조직 | 파일럿 후보, 업무 표준화, 확산 체계 | 업무 후보 발굴부터 파일럿 설계 패키지까지 연결한다. | 3일·1주 과정 안내, 파일럿 전환 기준 |
| IT·플랫폼 조직 | 구현 가능성, 보안, 검증, 유지보수 | 현업 요구사항을 `SKILL.md`, Orchestrator, 검증 체계로 전환한다. | 개발자 트랙, 참조 레포 구조, Security Notes |
| 보안·컴플라이언스 | 민감정보, 승인, 감사 가능성 | PII, 로그, 원본 보호, 승인 기록을 교육 산출물에 포함한다. | 보안 체크리스트, Workspace 규칙, Runbook |
| 현업 팀 리더 | 반복 업무 감소, 팀 합의, 실행 가능성 | 작은 업무 1건을 선택해 Skill, Workflow, 검증 기준으로 정리한다. | 비개발자 1일/3일 과정, 워크북 예시 |

제안 단계에서 가장 먼저 확인할 질문은 다음과 같다.

```text
이 조직은 AI 도구 사용법을 배우려는 것인가,
아니면 반복 가능한 업무 자동화 운영 모델을 만들려는 것인가?
```

# 3. 프로그램 소개문

## 3.1 표준 소개문

하네스(Harness) · Agent Skills 기반 기업 AI 교육은 사내 AI 활용을 개인별 프롬프트 역량에만 의존하지 않고, 팀과 조직이 함께 관리할 수 있는 업무 절차와 개발 표준으로 정착시키는 교육 프로그램입니다.

비개발자 트랙은 반복 업무를 Skill, Workflow, Workspace 규칙으로 정리하여 자동화 가능한 요구사항을 만듭니다. 개발자 트랙은 그 요구사항을 `SKILL.md`, Orchestrator, Task/Sub-agent, `AGENTS.md`, Smoke Verification, 보안·로그 기준으로 연결합니다.

교육의 목표는 단순히 AI 도구를 잘 쓰는 개인을 만드는 것이 아닙니다. 교육 후에는 실제 파일럿 또는 내부 표준화 논의에 사용할 수 있는 업무 시나리오, Skill 초안, Workflow, Workspace 규칙, 검증 체크리스트, 개발자용 하네스 템플릿이 남습니다.

---

## 3.2 짧은 소개문

많은 조직이 AI 도구를 도입했지만, 실제 현장에서는 개인별 프롬프트, 산출물 위치 혼선, 승인 누락, 검증 부재로 인해 성과가 반복되지 않는 문제가 발생합니다.

이 교육은 AI 활용을 개인의 요령이 아니라 조직의 Skill, Workflow, Workspace, 검증 체계로 전환하여, 반복 가능한 AI 자동화 파일럿과 사내 표준을 만들 수 있도록 돕습니다.

---

## 3.3 고객 제안용 소개문

귀사의 AI 활용이 개별 사용자 생산성 향상 단계에 머물러 있다면, 다음 단계는 조직이 함께 재사용할 수 있는 업무 표준과 개발 하네스를 만드는 것입니다.

본 교육은 현업 담당자가 업무 절차와 승인·검토 기준을 Skill과 Workflow로 정리하고, 개발자가 이를 `SKILL.md`, Orchestrator, Task, `AGENTS.md`, 검증 파이프라인으로 전환할 수 있도록 설계되어 있습니다.

1일 과정은 공통 언어와 최소 초안을 만들고, 3일 과정은 파일럿 설계 패키지를 완성하며, 1주 과정은 조직 표준·거버넌스·운영 로드맵까지 포함합니다.

# 4. 왜 필요한가

## 4.1 문제 정의

많은 기업이 AI 도구를 빠르게 도입하고 있지만, 실제 업무 적용에서는 다음 문제가 반복됩니다.

| 문제 | 설명 | 결과 |
| --- | --- | --- |
| 개인별 프롬프트 의존 | 사람마다 요청 방식이 다름 | 결과 품질 편차 발생 |
| 산출물 위치 혼선 | 입력, 중간 결과, 최종본이 섞임 | 재현과 감사 어려움 |
| 승인·검토 누락 | AI 결과가 검토 없이 사용됨 | 운영·보안 리스크 증가 |
| 검증 기준 부재 | 맞는 결과인지 판단 기준이 없음 | 파일럿 확산 어려움 |
| 현업·개발 연결 단절 | 업무 요구가 구현 기준으로 전달되지 않음 | 자동화 구현 지연 |
| 보안 기준 불명확 | 민감정보와 로그 기준이 없음 | 컴플라이언스 위험 증가 |

---

## 4.2 전환 방향

| 기존 방식 | 전환 방향 |
| --- | --- |
| 개인 프롬프트 | 조직 Skill |
| 임시 요청 | 재사용 가능한 업무 매뉴얼 |
| 수동 파일 관리 | `_workspace` 기반 산출물 규칙 |
| 암묵적 검토 | 명시적 Workflow와 승인 게이트 |
| 결과 확인 없음 | 검증 체크리스트와 Smoke Verification |
| 개발자에게 말로 전달 | `SKILL.md`, Task, `AGENTS.md`로 전달 |

# 5. 교육의 핵심 개념

| 개념 | 비개발자 관점 | 개발자 관점 |
| --- | --- | --- |
| Skill | 업무 매뉴얼, SOP, 금지 사항 | `SKILL.md`, 트리거, 입력·출력 계약 |
| Orchestrator | 승인·검토가 포함된 Workflow | 단계 조립, context passing, failure handling |
| Task | 업무를 나눈 단계 | I/O가 명확한 실행 단위 |
| Sub-agent | 역할이 나뉜 보조 수행자 | 도구 권한과 컨텍스트가 제한된 전문 에이전트 |
| `_workspace` | 입력·중간·최종 파일 구분 규칙 | 파일 경로 계약, 원본 보호, audit trail |
| `AGENTS.md` | 해당 없음 또는 안내 문서 | 저장소 진입점, 읽을 순서, 금지 경로 |
| Verification | 검증 체크리스트 | Smoke, Regression, Log keys |

# 8. 과정별 제안 문구

## 8.1 1일 과정 제안 문구

### 한 줄 설명

```text
AI 자동화를 처음 시작하는 조직이 하루 안에 공통 언어를 맞추고, 업무 1건에 대한 Skill과 Workflow 초안을 만드는 도입 워크숍입니다.
```

### 상세 설명

1일 과정은 AI 자동화가 아직 막연한 조직에 적합합니다.  
수강자는 반복 업무 1건을 선택하고, 이를 Skill, Workflow, Workspace 규칙으로 정리합니다. 개발자 과정에서는 동일한 업무를 `SKILL.md`, Orchestrator, Task, `AGENTS.md`, Smoke Verification으로 연결합니다.

교육 후에는 내부 논의나 파일럿 검토에 사용할 수 있는 최소 산출물이 남습니다.

### 주요 산출물

- 비개발자: 업무 시나리오 1페이지, Skill 초안, Workflow, Workspace 규칙
- 개발자: `SKILL.md`, Orchestrator 초안, Task 노트, `AGENTS.md`, Smoke Verification 메모

---

## 8.2 3일 과정 제안 문구

### 한 줄 설명

```text
팀 단위 자동화 후보를 선별하고, 실제 파일럿으로 넘길 수 있는 업무·기술 설계 패키지를 완성하는 실무형 과정입니다.
```

### 상세 설명

3일 과정은 단순한 아이디어 발굴을 넘어, 파일럿 준비가 필요한 조직에 적합합니다.  
비개발자 트랙은 자동화 후보 3~5건을 우선순위화하고, 선택한 업무에 대해 Skill v1, 승인·예외 Workflow, Workspace 운영 규칙, 리스크 레지스터, 검증 기준을 작성합니다.
