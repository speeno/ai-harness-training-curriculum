# Runbook 예시

## 1. 운영 목적

`제조 품질 이슈 로그·생산 회의록` 결과물을 정기적으로 만들고 검토하는 운영 절차를 표준화합니다.

## 2. 실행 순서

1. `workspace/in/`에 비식별 입력을 저장합니다.
2. `prompts/03-run-output.md`로 초안을 생성합니다.
3. `prompts/04-smoke-verify.md`로 검증합니다.
4. 검토자가 승인 후 공유합니다.

## 3. 예외 처리

- 입력 누락: 산출물 생성 중단
- 민감정보 발견: 비식별 후 재실행
- 승인 필요: 최종 공유 전 담당 리더 확인
